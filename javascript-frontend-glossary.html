CSS selector combination

A CSS selector allows to select a specific part of the DOM to give it some style. For instance, make the background red for the class called "classA"

.classA {
  background-color: red;
}

You can do the same with ids (#classA)or with html elements ( p or h1 ).

It is also possible to combine selectors and I always forget how it works

  • Grouping: .class1, .class2, .class3 { ... } select those 3 classes.
  • Descendant: .class1 .class2 {...} select class2 when it is inside a class1.
  • Adjacent sibling: .class1 + .class2 select class2 when it comes right after class1 at the same level.
  • General sibling: .class1 ~ .class2 select class2 when at the same level than class1
  • Child: .class1 > .class2 select class2 when it comes right after class1 as a children.

See more in this Cheatsheet.

Declarative vs Imperative code

Imperative code: spell out each step of how you want something done. Like do a for loop. Say the how. More verbose / low level.

Declarative code: say what it is that you want done. Like use an array method like every.. Say the what. More Concise / high level.

Destructuring

A js expression that allows to unpack values from arrays or properties from objects into distinct variables. Full reference 

// destructuring an array:
const [a, b] = [10, 20]
console.log(b) // 20

// destructuring an object:
const obj = { c: 3, d: 4, e: 89 }
const { c, d } = obj
console.log(d) // 4

Note that you can also rename directly

const otherObj = {f: 78, g: 12};
const {f: newNameForF, g: newNameForG} = otherObj;
console.log(newNameForF); // 78

DNS

DNS stands for Domain Name System, often compared to the "phonebook of internet". Basically, we reach a website thanks to its domain name (like d3-graph-gallery.com). But browsers work with IP adresses (like 192.168.1.1). DNS does the translation from domain name to IP.

IP adress

IP stands for Internet Protocol. The IP adress is a unique number attributed to your device, used to exchange information on the web through protocols. Think of this IP address as your computer’s street address.

Every website has an IP adress. The DNS makes the link between the IP adress and the website domain name.

Most common type of IP is IPv4 (version 4). It looks like 192.168.5.50. 4 numbers separated by dots, each ranging from 0 to 255. At some points the number of available IPv4 was not enough, so IPv6 was created to handle more adresses.

Pure function

A function can do mapping (take an input, returns an output) or procedures (interact with other stuff). Pure function are always about mapping. For each set of input, there is an output. For instance, Math.max() is a pure function.

A pure function:

  • does not have any side effect. It can’t alter any external state.
  • given the same input, returns the same output.

If it makes sense to call a function without using its output, the function is NOT pure.

Pure functions form the fundation of functionnal programming

Be careful about Immutability:

Common mistake is to mutate on of the argument that is passed to the function. Since the object is mutated, some side effects can happen. The function is thus impure. Example:


        const addToCart = (cart, item, quantity) => {
        cart.items.push({
          item,
          quantity
        });
        return cart;
      };

Spread syntax ...

The spread syntax allows to merge or clone object and concatenate arrays. Full reference

→ Merge: Note that if a property exists in several objects, its final value will be the last observed:

const obj1 = { a: 1, b: 2 }
const obj2 = { a: 1, b: 4 }
console.log({ ...obj1, ...obj2 }) // { a: 1, b: 4 }

→ Clone: shallow cloning here.

let obj = { a: 2 };
let clone = { ...obj };

→ Concatenate: For arrays

let arr1 = [0, 1, 2];
let arr2 = [3, 4, 5];
let concatArray = [...arr1, ...arr2];
console.log(concatArray) // [0, 1, 2, 3, 4, 5]
